IDL

The purpose of this page is to record some of the IDL tips and tricks I've found.

Tips

  • GDL: The Free Version Of IDL
    In case you didn't know, there is a free open-source version of IDL called GDL (GNU Data Language).
    I use it a fair amount, it very nearly matches the functionality of IDL.
    Here is the website:
    http://gnudatalanguage.sourceforge.net/

  • Log Each IDL Session
    You can automatically log each IDL session in a file with the date and time followed by a string you enter.
    Let's say you start a new IDL session on May 6 2015 at 2:25 PM. It will prompt you for the string:
    Enter IDL journal sub-name:
    And you enter:
    general_work
    Then all of your input and output during that IDL session will be logged in a file named "20150506_1425_general_work.jrn"
    The key is to set it up in your IDL startup file.
    Here is the code you would add:
    add_to_IDL_startup_file

  • Making IDL Plots In Color
    Allow color in IDL X-window: DEVICE, DECOMPOSED=0
    Load rainbow+white color table: LOADCT, 39, /SILENT
    Set up some colors:
    black=0
    darkpurple=15
    purple=30
    blue=56
    lightblue=95
    green=150
    yellow=190
    orange=203
    red=254
    white=255
    Example plot: PLOT, xarray, yarray, COLOR=green

  • Optimal Histogram Binsize
    One way to automatically select a histogram bin size is using the Freedman-Diaconis rule.
    Wikipedia link
    Here is an IDL function I wrote to compute it:
    fdbinsize.pro
    Example usage: histo = HISTOGRAM(array, BINSIZE=FDBINSIZE(array), LOCATIONS=xbins)

  • Multiple Plots In One IDL Window
    To do this, just set !p.multi = [0, col, row].
    0 means start a fresh window with the next plot command.
    col is the number of columns of plots.
    row is the number of rows of plots.
    To go back to a single plot, set !p.multi = 0.
